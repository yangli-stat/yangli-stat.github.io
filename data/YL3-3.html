What is Statistical Testing?

A statistical test is a formal procedure used to determine whether there is enough evidence in a sample of data to support a claim about a population.

The general idea is:

Is the observed effect real, or could it have occurred simply by random chance?

Example

Suppose researchers want to determine whether the average systolic blood pressure of patients at a clinic differs from the standard reference value of 120 mmHg.

  • Null hypothesis \(H_0: \mu=120\) versus alternative hypothesis \(H_1: \mu\neq 120\).
  • Need to draw a sample of size \(n\) from the population and measure their systolic BP.
  • Select the test statistic \(\bar{x}\).
  • If \(\bar{x}\) is close to 120, then don’t reject \(H_0\). If \(\bar{x}\) is far away from 120, reject \(H_0\) and claim \(H_1\).
    • Rejection region: \(\bar{x}\) in the rejection region \(\Rightarrow\) rejection
    • \(p\)-value

The p-value

The p-value measures the probability of observing data as extreme as, or more extreme than, the sample data, assuming that the null hypothesis is true.

A small p-value gives evidence against the null hypothesis.

A common decision rule is:

\[ \text{Reject } H_0 \quad \text{if} \quad p < \alpha \]

For example, if \(\alpha = 0.05\), then:

  • if \(p < 0.05\), reject \(H_0\);
  • if \(p \geq 0.05\), fail to reject \(H_0\).

Example: Blood Pressure Study

Suppose researchers want to test whether the average systolic blood pressure in a population differs from 120 mmHg.

The hypotheses are:

\[ H_0: \mu = 120 \]

\[ H_a: \mu \neq 120 \]

Suppose a sample is collected and a one-sample t-test gives:

\[ p = 0.012 \]

Since

\[ 0.012 < 0.05, \]

we reject \(H_0\).

Therefore, we conclude that there is statistically significant evidence that the mean systolic blood pressure differs from 120 mmHg.

R Example

The following code simulates systolic blood pressure data and performs a one-sample t-test.

set.seed(123)

blood_pressure <- rnorm(
  n = 30,
  mean = 125,
  sd = 12
)

t.test(
  blood_pressure,
  mu = 120
)

Testing CSR with the Quadrat Method

Given a point pattern, the first question is often whether there is any evidence to allow rejection of the null hypothesis of complete spatial randomness (CSR).

The quadrat method partitions the study region into \(r\) rows and \(c\) columns, which define \(m=rc\) non-overlapping subregions of equal area.

Null hypothesis \(H_0: \text{CSR}\) and alternative hypothesis \(H_1: \text{not CSR}\).

The test statistic is

\[ X^2 = \sum_{i=1}^m \frac{(\mbox{observed}_i - \mbox{expected})^2}{\mbox{expected}} = \sum_{i=1}^m \frac{(n_i - n^*)^2}{n^*}. \]

Under \(H_0\), \(X^2\) follows a chi-squared distribution with degrees of freedom \(m-1\), \(X^2\sim \chi^2_{m-1}\).

  • If \(X^2\) is too big, the pattern is aggregated.
  • If \(X^2\) is too small, the pattern is regular.
  • Otherwise, the pattern is CSR.

Quadrat Method in R

Generating a Point Pattern

suppressPackageStartupMessages({
  library(ggplot2)
  library(spatstat.geom)
  library(spatstat.explore)
})

set.seed(1)

n_random <- 100

random_df <- data.frame(
  x = runif(n_random, 0, 1),
  y = runif(n_random, 0, 1),
  pattern = "Random"
)

ggplot(random_df, aes(x = x, y = y)) +
  geom_point(size = 2, alpha = 0.8) +
  coord_equal(xlim = c(0, 1), ylim = c(0, 1)) +
  labs(x = "x", y = "y") +
  theme_minimal()

Convert to a ppp Object

random_ppp <- ppp(
  x = random_df$x,
  y = random_df$y,
  window = owin(
    xrange = c(0, 1),
    yrange = c(0, 1)
  )
)
random_ppp
## Planar point pattern: 100 points
## window: rectangle = [0, 1] x [0, 1] units

A ppp object in spatstat means a planar point pattern object. It is the main data format that spatstat uses for spatial point pattern analysis. A ppp object stores:

  • the \(x\)-coordinates of the points,
  • the \(y\)-coordinates of the points,
  • the observation window (owin),
  • optional marks, such as point type or group label.

Quadrat Test for CSR

On average, each quadrat has \(100/16=6.25\) dots. The test statistic

\[ X^2=\sum_{i=1}^{16}\frac{(O_i-6.25)^2}{6.25}=15.2 \]

q_test <- quadrat.test(random_ppp, nx = 4, ny = 4)
q_test
## 
##  Chi-squared test of CSR using quadrat counts
## 
## data:  random_ppp
## X2 = 15.2, df = 15, p-value = 0.8742
## alternative hypothesis: two.sided
## 
## Quadrats: 4 by 4 grid of tiles

The \(p\)-value is large so we won’t reject \(H_0\). This means there is not strong evidence against complete spatial randomness.

The point pattern is generated with completely spatial randomness. It is not surprising that we don’t reject CSR.

Quadrat Test on an Aggregated Pattern

set.seed(1)
n_clusters <- 10
points_per_cluster <- 10

cluster_centers <- data.frame(
  cx = runif(n_clusters, 0.05, 0.95),
  cy = runif(n_clusters, 0.05, 0.95)
)

aggregated_df <- do.call(
  rbind,
  lapply(1:n_clusters, function(k) {
    data.frame(
      x = rnorm(points_per_cluster, cluster_centers$cx[k], sd = 0.05),
      y = rnorm(points_per_cluster, cluster_centers$cy[k], sd = 0.05),
      cluster = k
    )
  })
)

aggregated_df$x <- pmin(pmax(aggregated_df$x, 0), 1)
aggregated_df$y <- pmin(pmax(aggregated_df$y, 0), 1)
aggregated_df$pattern <- "Aggregated"

ggplot(aggregated_df, aes(x = x, y = y)) +
  geom_point(size = 2, alpha = 0.8) +
  coord_equal(xlim = c(0, 1), ylim = c(0, 1)) +
  labs(
    title = "Aggregated Spatial Point Pattern",
    x = "x",
    y = "y"

  theme_minimal()

  x = aggregated_df$x,

)

Alternative Hypothesis

By default, quadrat.test tests \(H_0: \text{CSR}\) versus \(H_a: \text{not CSR}\). If \(H_0\) is rejected, we only know it’s not CSR.

To be more specific, we can use one-sided test.

  • alternative = "two.sided" to test \(H_0: \text{CSR}\) versus \(H_1: \text{not CSR (regular or aggregated)}\).
  • alternative = "regular" to test \(H_0: \text{CSR}\) versus \(H_1: \text{regular}\).
  • alternative = "clustered" to test \(H_0: \text{CSR}\) versus \(H_1: \text{aggregated}\).

Using the regular pattern example on the previous page,

one_side_test1 <- quadrat.test(regular_ppp, nx = 4, ny = 4, alternative = "regular")
one_side_test1
## 
##  Chi-squared test of CSR using quadrat counts
## 
## data:  regular_ppp
## X2 = 4.32, df = 15, p-value = 0.003514
## alternative hypothesis: regular
## 
## Quadrats: 4 by 4 grid of tiles

CSR is rejected and we accept a regular pattern.

If we want to test CSR against aggregated, use alternative = "clustered".

one_side_test2 <- quadrat.test(regular_ppp, nx = 4, ny = 4, alternative = "clustered")
one_side_test2
## 
##  Chi-squared test of CSR using quadrat counts
## 
## data:  regular_ppp
## X2 = 4.32, df = 15, p-value = 0.9965
## alternative hypothesis: clustered
## 
## Quadrats: 4 by 4 grid of tiles

The pattern is not aggregated.
